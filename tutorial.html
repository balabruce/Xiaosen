Chapter 01

什麼是 Claude Code？

Claude Code 是 Anthropic 推出的 AI 程式設計助理工具，讓你可以在終端機（Terminal）裡直接跟 Claude 對話，請它幫你寫程式、修 Bug、解釋程式碼，甚至整個專案的架構都能一起討論。

跟一般 AI 聊天有什麼不同？

❌ 一般 AI 聊天（如 ChatGPT 網頁）
  • 貼程式碼、複製回來，很麻煩
  • AI 看不到你的檔案結構
  • 要手動執行每個指令
  • 沒辦法直接修改你的程式
✅ Claude Code
  • 直接在你的專案資料夾工作
  • 能讀取、修改、新增任何檔案
  • 可以自己執行 Terminal 指令
  • 記住整個對話脈絡

用一句話形容

Claude Code 就像你多了一個 超厲害的工程師朋友坐在旁邊，你說「幫我改這個功能」，他直接拉開你的程式碼開始動工，不需要你解釋上下文、不需要你複製貼上。

💡
小森的例子：這個網頁版的小森（index.html）本身就是用 Claude Code 開發的。從設計聊天界面、串接 Anthropic API、到加上記憶功能，全程透過對話完成。

Chapter 02

安裝 Claude Code

事前準備

1
Node.js（版本 18 以上）

到 nodejs.org 下載安裝。安裝後打開 Terminal 輸入 node -v 確認版本。

2
Anthropic API Key

到 console.anthropic.com 註冊帳號，創建 API Key。這是 Claude Code 的「通行證」。

3
Terminal（終端機）

Mac 用內建的 Terminal 或 iTerm2，Windows 用 Windows Terminal 或 PowerShell。

安裝指令

bash 
# 全域安裝 Claude Code CLI
npm install -g @anthropic-ai/claude-code

# 確認安裝成功
claude --version

設定 API Key

bash 
# 方法一：第一次啟動時 Claude Code 會引導你輸入
claude

# 方法二：設定環境變數（長期有效）
# macOS / Linux
echo 'export ANTHROPIC_API_KEY="sk-ant-api03-你的key"' >> ~/.zshrc
source ~/.zshrc

# Windows PowerShell
[System.Environment]::SetEnvironmentVariable("ANTHROPIC_API_KEY", "sk-ant-api03-你的key", "User")
🔒
API Key 安全提醒：千萬不要把 API Key 直接寫在程式碼裡或上傳到 GitHub。用環境變數的方式存放最安全。

Chapter 03

第一次啟動 Claude Code

進入專案資料夾，啟動 Claude Code

bash 
# 移動到你的專案資料夾
cd /你的專案路徑

# 例如：開發小森這個專案
cd ~/projects/xiaosen

# 啟動 Claude Code（進入互動模式）
claude

你會看到什麼

啟動後進入互動模式，畫面大概長這樣：

C
✻ Welcome to Claude Code!

/help for help, /status for status

cwd: /Users/you/projects/xiaosen
你
這個專案是做什麼的？幫我簡單說明一下
C
這是一個名為「小森」的 AI 聊天網頁應用，是一個純前端的單頁應用（index.html）。

主要功能：
• 串接 Anthropic Claude API 進行對話
• 透過 Supabase 儲存對話記錄和記憶
• 支援圖片上傳和網路搜尋

整個應用只有一個 HTML 檔案，所有的樣式和 JavaScript 都內嵌其中。
✅
Claude Code 最強大的地方是你不需要先解釋專案結構。它會自己讀取當前資料夾的所有檔案，立刻就理解整個專案。

單次執行（不進入互動模式）

bash 
# 直接執行一個問題就結束
claude "幫我解釋 index.html 這個檔案的結構"

# 或是用 -p 旗標
claude -p "列出這個專案所有的函式名稱"

Chapter 04

互動模式中的核心指令

在 Claude Code 的互動模式裡，有一些以 / 開頭的內建指令：

最常用指令

/help

顯示所有可用指令的說明

/clear

清除對話記錄，重新開始（省 Token）

/status

查看目前狀態、消耗的 Token 數

/exit

離開 Claude Code（或按 Ctrl+C）

/model

切換使用的 AI 模型

/cost

查看這次對話花了多少費用

/config

開啟設定選單

/review

查看最近的程式碼修改

鍵盤快捷鍵

按鍵 功能
Enter 送出訊息
Shift + Enter 換行（不送出）
↑ / ↓ 瀏覽歷史訊息
Ctrl + C 中斷目前執行中的任務
Ctrl + D 離開 Claude Code
Esc 取消正在輸入的訊息

執行模式旗標

bash 
# 不需要確認，直接執行所有操作（小心使用）
claude --dangerously-skip-permissions

# 只閱讀，不修改任何檔案
claude --no-file-operations

# 指定使用特定模型
claude --model claude-opus-4-8

# 把對話記錄儲存到檔案
claude --output-format json > session.json

Chapter 05

怎麼說話 Claude 才聽得懂？

提示詞（Prompt）的品質直接影響結果好不好。以下是最實用的幾個原則：

原則一：說清楚你要什麼

❌ 太模糊
  • 「幫我改一下這個」
  • 「程式壞了怎麼辦」
  • 「讓它更好」
✅ 明確具體
  • 「把 send() 函式的錯誤訊息改成繁體中文」
  • 「index.html 第 437 行有 undefined 錯誤，幫我找原因」
  • 「讓送出按鈕在訊息為空時變灰色且無法點擊」

原則二：給予上下文

你
這個專案是一個 AI 聊天 App（小森）。我想在聊天泡泡上加一個長按複製功能，讓用戶可以長按訊息來複製文字。請幫我實作。
C
我來幫你在 bubble 元素上加長按複製功能。我會使用 contextmenu 和 touchstart 事件來同時支援桌面右鍵和手機長按…

原則三：分步驟處理複雜任務

🧠
一次叫 Claude 做太多事，品質會下降。把大任務拆開：先叫他「分析」，再叫他「設計」，最後「實作」。
你
我想加一個深色模式切換功能。先幫我分析目前的 CSS 架構，不要動程式碼。
C
目前使用 CSS 變數（:root）管理所有顏色，這非常適合深色模式。主要變數有 --bg、--paper、--user-bubble 等 8 個…
你
好，現在幫我加深色模式，在 header 加一個切換按鈕。

原則四：說明限制條件

提示詞範例 
"幫我優化 fetchMemories 函式的效能，
但不要改變它的 API 介面（函式簽名不能變），
也不要引入新的外部套件。"

"為 send() 函式加上錯誤處理，
只處理 HTTP 4xx 和 5xx 錯誤，
網路斷線的情況先不用處理。"

常見任務的提示詞模板

任務類型 提示詞模板
修 Bug 「[檔案名] 的 [函式/功能] 出現 [錯誤訊息]，幫我找出原因並修復」
新增功能 「幫我在 [位置] 加上 [功能描述]，需要符合 [限制條件]」
重構程式碼 「[函式/檔案] 太複雜了，幫我重構，保持原有功能不變」
解釋程式碼 「解釋 [函式/模組] 的運作邏輯，用白話文說明」
寫測試 「為 [函式] 寫單元測試，需要涵蓋正常情況和邊界案例」

Chapter 06

實用工作流程

日常開發流程

1
進入專案，啟動 Claude Code

cd 你的專案 && claude

2
先說明今天要做什麼

「今天我要加深色模式，先讓我解釋一下需求…」

3
叫 Claude 分析、規劃

「根據需求，你覺得要修改哪些地方？列出計劃，先不要動程式碼」

4
逐步確認、實作

「第一步先加 CSS 變數，其他的等我確認後再繼續」

5
測試、修正、提交

「幫我用 Git 提交這些修改，commit message 用繁體中文」

管理 CLAUDE.md

在專案根目錄創建 CLAUDE.md 檔案，可以告訴 Claude 這個專案的重要規則，每次啟動都會自動讀取。

CLAUDE.md 
# 小森專案規則

## 技術架構
- 純前端，單一 HTML 檔案（index.html）
- 使用 Anthropic Claude API（anthropic-version: 2023-06-01）
- 後端儲存用 Supabase（REST API）

## 程式碼規範
- 所有 UI 文字用繁體中文
- 不引入外部 JavaScript 套件
- CSS 變數定義在 :root，不要 hardcode 顏色
- 錯誤訊息要讓一般使用者看得懂

## 不能做的事
- 不能修改 API endpoint（永遠用 api.anthropic.com）
- 不能移除 localStorage 相容性
- 不能破壞現有的記憶功能

控制費用的技巧

💰
Token 費用管理：
• 用 /clear 定期清除對話，避免 context 太長
• 分析/閱讀任務用 Sonnet，複雜設計/架構用 Opus
• 用 /cost 隨時查看花費
• 不需要 Claude 執行操作時，改用 --no-file-operations 旗標

Chapter 07

實戰練習：用小森專案練手

用這個小森專案（你現在看的這個網頁的 index.html）來練習：

初級：讀懂程式碼

試著對 Claude Code 輸入以下這些問題，看它能不能回答：

練習指令
"index.html 的 send() 函式做了哪些事？用步驟說明"

"SYSTEM 這個常數是什麼？它在哪裡被用到？"

"記憶功能的流程是什麼？從用戶送訊息到儲存記憶，整個流程說明一下"

"這個專案用了哪些 Anthropic API 的功能？"
🎓
目標：能讓 Claude 解釋你不懂的程式碼段落。

中級：修改功能

練習指令
"在聊天界面的右上角加一個『清除對話』按鈕，點了就清空畫面上的訊息"

"把 header 的狀態文字改成顯示『已連線 · Sonnet · 🔍搜尋開啟』這樣的格式"

"在 addMsg 函式加上防 XSS 處理，確保訊息內容不會被當 HTML 執行"

"幫我把 needsSearch 函式的關鍵字清單改成可以在設定裡客製化"

進階：獨立開發功能

練習指令
"幫我加一個對話匯出功能，可以把聊天記錄下載成純文字檔"

"幫我實作訊息搜尋功能，可以在對話裡全文搜尋"

"我想讓小森能記住用戶的語氣偏好（正式/輕鬆），並自動調整說話方式，幫我設計這個功能"
🏆
進階練習不只是「叫 Claude 做」，而是你要跟 Claude 一起討論設計，再讓它實作。試著問「有哪些方法可以做到？各自的優缺點是什麼？」
